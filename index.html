FAQ — DOM Concepts

What is the DOM?

+

The Document Object Model is the browser's live, in-memory map of your HTML — translated from static markup into a tree of programmable objects that JavaScript can read and mutate.

Why use addEventListener?

+

addEventListener decouples your logic from your markup, allows multiple handlers on one element, and provides a clean reference to remove the handler later — unlike inline onclick attributes.

const vs let — when to use which?

+

Use const by default for DOM references and values that never reassign. Use let only when the value must mutate (counters, loop variables). Avoid var — it is function-scoped and prone to hoisting bugs.

Why prefer textContent over innerHTML?

+

textContent treats the value as plain text, preventing Cross-Site Scripting (XSS) attacks. innerHTML parses string as HTML — injecting user-supplied data this way is a critical security vulnerability.
